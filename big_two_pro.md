# 🎴 專案筆記：大老二專業 AI 鐵律版 (Big Two Pro)

## 1. 專案概述
本專案是一個基於網頁技術（HTML/CSS/JS）開發的單機版大老二遊戲。除了完整的牌型判定與遊戲流程外，特別強化了**傳統鐵律約束**與**中階 AI 策略**。

*   **檔案名稱：** `big_two_pro.html`
*   **運行環境：** 任何現代網頁瀏覽器 (Chrome, Edge, Firefox)
*   **核心技術：** Vanilla JavaScript, CSS3 (Flexbox/Transitions), HTML5

## 2. 核心規則實作 (The Iron Rules)
### 2.1 梅花 3 鐵律 (Club 3 First)
*   **起始判定：** 遊戲開始時，系統會掃描所有玩家手牌，持有梅花 3 (`power: 0`) 的玩家強制獲得首輪出牌權。
*   **首抽約束：** 第一局的第一手牌**必須包含梅花 3**，否則系統會攔截並彈出警告，確保符合正規大老二玩法。

### 2.2 牌型權重判定
系統採用 `comboRank` 等級制來判定五張牌組的強度：
1.  **同花順 (Straight Flush)** - `Rank: 5` (最高)
2.  **鐵支 (Four of a Kind)** - `Rank: 4`
3.  **葫蘆 (Full House)** - `Rank: 3`
4.  **同花 (Flush)** - `Rank: 2`
5.  **順子 (Straight)** - `Rank: 1`
*單張與對子屬於基礎牌型，僅比對最大張的 `power` 值。*

## 3. AI 策略邏輯
AI 具備「全牌型掃描」能力，其思考邏輯如下：
*   **主動攻擊 (自由出牌)：** 優先嘗試打出最強的五張牌型（五張 > 對子 > 單張），以利消耗手中雜牌並保持主導權。
*   **被動防守 (壓牌)：**
    *   會進行「越級壓牌」：例如玩家打順子，AI 若有葫蘆會直接壓過。
    *   **權力保留：** AI 僅會選擇手中剛好大過桌面的最小組合，保留大牌作為最後衝刺。

## 4. 介面與交互優化 (UX/UI Highlights)
*   **滑鼠鎖定容器 (Card Slot)：** 透過固定位置的 Slot 包裹位移的卡片，解決了 CSS Hover 時因元素移動導致的「閃爍/跳動」問題。
*   **牌桌狀態分離：** 將「牌組顯示區」與「文字訊息區」分離。當有人 Pass 時，桌面的牌組會完整保留，直到三家皆 Pass 才正式清除，方便玩家判斷戰況。
*   **渲染延遲判定：** 在 `checkWin` 邏輯中加入 600ms 延遲，確保最後一張獲勝牌能被玩家「看見」後才彈出勝負視窗。

## 5. 關鍵技術修正記錄
| 問題描述 | 解決方案 |
| :--- | :--- |
| 選牌時鼠標懸停會導致牌面快速跳動 | 引入 `.card-slot` 固定容器鎖定滑鼠焦點 |
| 最後一張牌打出後立刻消失 | 加入 `setTimeout` 延遲 `alert` 彈窗 |
| 點擊 Pass 後桌面的牌會被清空 | 將 `table-cards-container` 分離獨立控制 |
| 牌桌上的五張牌重疊在一起 | 針對牌桌專屬 CSS 取消 `absolute` 定位並設定 `relative` 展開 |
